# Contribuir a TOONBench.jl

Gracias por tu interés en contribuir a TOONBench.jl

## Proceso de Contribución

### 1. Fork y Clone

```bash
git clone https://github.com/ibarvivas/TOONBench.jl.git
cd TOONBench.jl
```

### 2. Crear Rama

```bash
git checkout -b feature/mi-nueva-funcionalidad
```

### 3. Desarrollar

- Escribe código limpio y documentado
- Sigue el estilo de código existente
- Añade tests si es posible
- Actualiza documentación según corresponda

### 4. Commit y Push

```bash
git add .
git commit -m "Descripción clara del cambio"
git push origin feature/mi-nueva-funcionalidad
```

### 5. Pull Request

- Abre PR en GitHub
- Describe qué cambia y por qué
- Referencia issues relacionados

## Estilo de Código

### Julia

- Nombres descriptivos en inglés
- Documentación inline en español
- Docstrings para todas las funciones públicas
- Indentación: 4 espacios
- Líneas máximo 92 caracteres

### Documentación

- Comentarios exhaustivos en español
- Ejemplos prácticos en docstrings
- README actualizado para cambios significativos

## Áreas de Contribución

### Prioridad Alta

- Soporte para más tipos de datos
- Integración con APIs LLM (OpenAI, Anthropic)
- Optimización de performance
- Tests unitarios

### Prioridad Media

- Dashboard interactivo con Pluto.jl
- Soporte para DataFrames.jl directo
- Exportación a más formatos
- Benchmarks adicionales

### Prioridad Baja

- Traducciones
- Temas visuales
- Integraciones adicionales

## Reportar Bugs

Usa GitHub Issues con:

- Descripción clara del problema
- Pasos para reproducir
- Versión de Julia y TOONBench
- Output de error completo

## Preguntas

Para preguntas, usa:
- GitHub Discussions
- Issues con label "question"

## Código de Conducta

Se amable, respetuoso y constructivo.
